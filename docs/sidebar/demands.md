# 侧边栏

## 需求

请你根据以下具体需求，/sop-plan 规划出文档，并保存到 @docs/sidebar/ 目录下给我审核。

具体需求：
1. 在首页新增侧边栏，侧边栏顶部有 3 个按钮：（1）新增文件；（2）新增目录；（3）递归展开/收起目录。均使用 lucide 图标展示，无需使用文字。
2. 侧边栏顶部下面展示文件和目录结构，并使用 `localforage` 这个 npm 包做本地存储，保存对象为文件、目录、文件内容、当前打开文件 id。我理解需要建立对应的数据结构来映射文件和目录，这个你自己规划一下。
3. 侧边栏的文件目录树最好能使用现成的 npm 包，要求性价比高、稳定的，下载使用量大的，需要能支持拖拽文件或者目录位置。

要求：
1. 先了解已有的代码架构和逻辑
2. 然后才做调研规划

### 反馈

以下是审核意见：
1. 侧边栏树中的文件夹图标不要设置为蓝色，默认颜色即可，不需要高亮区分
2. 点击侧边栏树中的文件夹并不能打开或者收起，需要支持上
3. 将文件拖进文件夹并刷新之后，文件又回到原位，此外选中文件夹之后，再点击新建文件图标，新建出来的文件并没有在对应的文件夹中
4. 新建文件之后，文件内容应该是空白的才对
