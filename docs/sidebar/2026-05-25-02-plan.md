# 侧边栏右键菜单 — 方案规划

## 需求

在侧边栏文件树中，右键点击文件或目录节点时弹出上下文菜单，提供快捷操作入口。菜单项根据节点类型（文件 vs 目录）不同而变化。

## 菜单项设计

### 文件节点右键菜单

```
┌─────────────────────┐
│ 📄 Open             │  仅当不是当前打开文件时显示
├─────────────────────┤
│ ✏️  Rename          │  触发内联重命名（复用现有点击逻辑）
├─────────────────────┤
│ 📄 New File         │  在当前文件同级目录下新建文件
│ 📁 New Folder        │  在当前文件同级目录下新建目录
├─────────────────────┤
│ 🗑️  Delete          │  删除文件
└─────────────────────┘
```

### 目录节点右键菜单

```
┌─────────────────────┐
│ 📄 New File         │  在该目录下新建文件
│ 📁 New Folder        │  在该目录下新建子目录
├─────────────────────┤
│ ✏️  Rename          │
├─────────────────────┤
│ ⊞ Expand All       │  仅当有子节点且有关闭的目录时显示
│ ⊟ Collapse All     │  仅当有子节点且有展开的目录时显示
├─────────────────────┤
│ 🗑️  Delete          │  删除目录及其所有内容
└─────────────────────┘
```

## 技术方案

### 不引入新依赖，纯 vanilla JS 实现

理由：
- 右键菜单逻辑简单（创建 DOM → 定位 → 销毁），~100 行代码即可完成
- 现有代码库已经在用相同的 vanilla JS 模式（`tree-renderer.js` 的手写 DOM 渲染）
- 避免引入低质量/过度设计的 npm 包

### 新增文件：`src/context-menu.js`

职责：纯 UI 层，负责菜单 DOM 的创建、定位、销毁，不包含业务逻辑。

对外暴露：

```js
// 显示菜单，返回 Promise，用户选择后 resolve 所选 action key
showContextMenu(x, y, items, theme) → Promise<string|null>
// 手动关闭菜单（点击外部、按 Escape 时调用）
closeContextMenu()
```

`items` 数据结构：

```ts
interface MenuItem {
  key: string;          // 唯一标识，如 'rename', 'delete'
  label: string;        // 显示文字
  icon: IconNode;      // lucide 图标节点
  disabled?: boolean;   // 是否置灰
  danger?: boolean;     // 是否标红（删除操作）
}
```

### 修改文件：`src/tree-renderer.js`

- `renderTree()` 中新增 `contextmenu` 事件监听器，与已有的 `click`/`dblclick` 并行管理（同样需要清除旧监听器）
- 新增 callback：`onContextMenu(nodeId, x, y)` → 传递给 `sidebar.js` 处理

### 修改文件：`src/sidebar.js`

- `refreshTree()` 中新增 `onContextMenu` 回调实现：
  1. 根据 `nodeId` 和节点类型构造 `MenuItem[]`
  2. 调用 `showContextMenu(x, y, items, theme)`
  3. 根据 Promise resolve 的 `key` 分发到对应的处理函数

### 修改文件：`public/css/style.css`

新增约 50 行上下文菜单样式 + 暗色主题适配。

### 修改文件：`src/constants.js`

新增 CSS 选择器和 class 常量。

## 菜单定位逻辑

```
┌─────────┬─────────────────────────────
│ Sidebar │
│         │
│    ┌────┴────┐  优先在鼠标右下方展开
│    │  Menu   │
│    └─────────┘  若超出视口底部 → 向上展开
│                 若超出视口右侧 → 向左偏移
└──────────────────────────────────────
```

使用 `getBoundingClientRect()` 计算视口边界，在菜单 DOM 创建后、显示前做一次位置修正。

## 关闭菜单的时机

| 事件 | 行为 |
|---|---|
| 点击菜单项 | 执行操作 + 关闭 |
| 点击菜单外部任意位置 | 关闭 |
| 按 Escape | 关闭 |
| 窗口 resize | 关闭 |
| 滚动侧边栏 | 关闭 |

## 暗色主题

菜单 DOM 创建时不依赖 CSS `data-theme` 选择器，改为直接传入当前 `theme` 参数（`'dark'` | `'light'`），在 JS 中给菜单容器设置对应的 class 或内联样式。这避免了菜单作为 `document.body` 的直接子元素时脱离 `[data-theme]` 作用域的问题。

## 菜单项图标

使用 lucide 图标，通过 `createElement(Icon)` 创建 SVG 节点：

| 菜单项 | lucide 图标 |
|---|---|
| Open | `FileText` |
| Rename | `Pencil` |
| New File | `FilePlus` |
| New Folder | `FolderPlus` |
| Expand All | `ChevronsDownUp` |
| Collapse All | `ChevronsUpDown` |
| Delete | `Trash2` |

## 实施步骤

1. 创建 `src/context-menu.js` → 实现 `showContextMenu` / `closeContextMenu`
2. 修改 `src/tree-renderer.js` → 新增 `contextmenu` 事件 + `onContextMenu` callback
3. 修改 `src/sidebar.js` → 实现 `onContextMenu`，构造菜单项，分发 action
4. 修改 `public/css/style.css` → 菜单样式 + 暗色适配
5. 构建验证 + 手动测试

预计改动量：~100 行新增（context-menu.js）+ ~30 行修改（tree-renderer.js）+ ~60 行修改（sidebar.js）+ ~50 行新增（CSS）。
